# 小学语文课件比较评估

## 目标与边界

依据本地权威资料比较两份课件文本大纲的教学质量。只评价大纲可支持的教学设计与内容安排，不根据文本推断美术、动画、现场执教效果或学生真实学习结果。

默认资料根目录为 `D:\语文PPT`。用户只需给出课文名；不要要求用户重复上传已存在于该目录中的资料。

## 工作流程

### 1. 定位课文文件夹

1. 从用户话语提取课文标题，容许 `漏`、`《漏》`、`26《漏》` 或包含年级册次的表达。
2. 在 `D:\语文PPT` 下递归查找目录名包含该课文标题且目录中存在 `.txt` 文件的候选课文文件夹。
3. 优先使用用户明确指定的年级、册次或课序筛选候选目录，并从父目录记录年级册次。
4. 若只命中一个文件夹，直接继续；若命中多个，列出年级册次与目录名，请用户选定；若未命中，报告检索根目录与标题，并请用户核对存放位置或标题。

可使用 PowerShell 进行查找，例如将 `$title` 替换为提取出的纯课文名：

```powershell
$root = 'D:\语文PPT'
$title = '漏'
Get-ChildItem -LiteralPath $root -Directory -Recurse |
  Where-Object { $_.Name -like "*$title*" -and (Get-ChildItem -LiteralPath $_.FullName -File -Filter '*.txt' -ErrorAction SilentlyContinue) }
```

### 2. 识别材料文件

读取选定课文文件夹中的 `.txt` 文件清单，并按文件名与内容用途分类：

- 教学依据资料：文件名含 `教材`、`原文`、`课后题`、`教师用书`、`教参`、`教学设计`、`教学指导` 等。一个综合资料文件可以同时承担多类依据。
- 待评课件资料：文件名含 `同行`、`课件`、`PPT` 或 `ppt`，且不是教学依据资料。
- 未明确归类的文件：先查看开头和结构，再判断用途；仍不明确时向用户确认。

仅在找到足以建立教学基准的依据资料，并明确识别出两份待评课件时自动评分。若待评课件超过两份，列出文件名请用户指定两份；少于两份或缺少关键依据时，列明缺项，不生成伪精确总分。

在报告开头写明实际读取的年级册次、课文文件夹及三类文件清单。将两份课件按原文件名标记为 `课件 A` 和 `课件 B`。

### 3. 建立本课教学评价基准

先读教学依据资料，再读待评课件。按以下优先级提炼基准：

1. 教材原文、课后题、生字词、学习提示和单元语文要素。
2. 教师用书明确说明的编排意图、教学目标、重难点与教学建议。
3. 教学设计或教学指导中可落实上述要求的活动组织建议。

将评价基准整理为简明清单，包括：必须完成的学习任务、重点难点、建议的语文实践方式、课时安排约束及可能需要核对的事项。资料冲突时指出冲突，并以教材和教师用书能够支持的安排为准。

### 4. 分别审读课件并比较

读取 [references/evaluation-rubric.md](references/evaluation-rubric.md)，按八个维度为 A/B 评分。每个维度必须写出：

- A 得分、B 得分。
- 来自教学依据资料的评价要求。
- 课件大纲中支撑得分或扣分的具体环节、页面主题、题目或活动证据。
- 分差理由；无实质差异时明确写明。

不能因页数多、活动热闹、装饰性描述丰富而直接提高评价。课件若加入资料无法支持的答案、知识结论或目标，作为准确性和教学风险问题指出。

### 5. 输出比较报告

读取 [references/output-template.md](references/output-template.md)，按其结构交付：

1. 检索结果与评价边界。
2. 本课教学评价基准。
3. A/B 分项评分表与总分。
4. 胜出者及关键证据。
5. 课件 A 诊断与改进建议。
6. 课件 B 诊断与改进建议。
7. 可融合之处、共同风险与待人工核对事项。

如两份课件总体差异不足以稳定判胜，可给出“基本持平”或“各有优势”，说明影响判断的资料限制或主要权衡，而不是强行指定赢家。

## 忠实性要求

- 只依据已读取文件评价；不要凭常识补写某课的教材要求、原句或教参结论。
- 结论以文件名和可核查内容为证据，必要时引用短语或概括对应环节。
- 对课文原句、生字读音、课后题表述及教材目标中的不确定内容标记“需回源核对”。
- 不将同行课件视为权威依据；两份课件都是被评价对象。
- 文本大纲不能支持视觉效果评价时，明确排除该项，不凭想象加减分。

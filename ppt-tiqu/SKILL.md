# 同行课件逐页提取

## 目标

将同行课件转换为忠实、可核对、保留页界的参考大纲。只提取同行课件实际存在的内容并总结其整体组织方式，不将同行文案直接改写为用户自己的成品课件。

## 输入优先级

按以下顺序接受材料：

1. 优先处理 `.pptx` 源文件，因为页界清楚且通常能提取原生文字。
2. 处理 `.pdf` 课件时，严格按 PDF 页码逐页读取；不要跨页合并。
3. 处理 `.png`、`.jpg` 等按页截图时，按文件名顺序或用户指定顺序视为页序，并在开头声明页序依据。

当用户一次提供多套同行课件时，先分别输出每套逐页大纲，再在用户要求比较时输出跨课件结构摘要。不要把多套课件页面混在一个页序中。

## 文件保存位置

提取完成后，除在对话中报告结果外，必须将最终逐页提取大纲保存为 UTF-8 编码的 `.txt` 文件。默认保存位置不再固定到某个资料库，而是保存到输入课件所在文件夹：

- 输入是 `.pptx` 或 `.pdf` 文件时，TXT 保存到该文件所在文件夹。
- 输入是一组按页图片时，TXT 保存到这些图片所在文件夹；若图片分散在多个文件夹，先询问用户选择保存位置。
- 用户明确指定输出目录或文件名时，优先遵从用户指定。
- 不要因为无法确认年级册次或课文名称而改存到当前工作目录；只有在输入文件路径不可写或无法确定来源文件夹时，才说明原因并请用户确认保存位置。
- 完成交付时明确告知用户最终 TXT 文件的完整路径。
## 提取流程

### 1. 建立页面清单

先确认文件名、文件格式、总页数及可判断的课文名称。无论何种格式，后续输出必须使用明确的 `第 X 页` 页面边界。

### 2. 优先读取原生文本

- 对 `.pptx`，优先逐张幻灯片读取可编辑文本，并保留每张幻灯片的文本范围。
- 对 `.pdf`，优先逐页提取文本，保留 PDF 原页码。
- 对按页图片，无原生文字可取，直接进入视觉识别流程。

不得仅依据一次整体转文本结果生成大纲，因为整体转换可能丢失页面边界或阅读顺序。

### 3. 按需使用 OCR 或视觉识别

OCR 指识别页面图片中可见文字。仅在以下情况下对对应页面启用：

- 页面本身是图片或扫描页。
- 原生提取为空，但页面画面明显含有文字。
- 原生文本出现乱码、缺失标题、缺失练习题或明显不完整。
- PPT 页面包含承载教学信息的截图、插图文字或拍照教材内容，原生提取无法覆盖。

处理规则：

- 对高风险页面先渲染或查看原页图像，再补充识别可见文字。
- 将提取方式标注为 `原生文本`、`OCR识别，需核对` 或 `原生文本 + OCR补充，需核对`。
- 不能确定的字词写为 `[无法辨认]`，不得根据语境猜写。
- 课文原句、生字、拼音、答案和题干即使识别清晰，也提醒后续以教材权威资料核对；同行资料只作为参考。

### 4. 判断页面类型和版式信息

根据页面中实际存在的内容，为每页选择适当页面类型，例如：

`封面`、`导入`、`作者介绍`、`学习目标`、`识字写字`、`词语`、`朗读指导`、`整体感知`、`课文讲解`、`重点句赏析`、`课堂活动`、`练习`、`拓展`、`总结`、`作业`、`其他`。

仅简述对原创大纲规划有帮助的视觉或版面信息，例如“左侧课文原句，右侧场景插图”“用流程图解释故事发展”。不要耗费篇幅记录纯装饰元素。

## 输出格式

### 交付文件格式与命名

- 最终逐页提取大纲保存为 UTF-8 编码的 `.txt` 文本文件，不默认以 `.md` 作为交付文件格式。
- 以课件中可识别的课文名称作为文件名主题；例如课文《漏》。
- 同一课文的第一份同行课件提取文件命名为 `《课文名》_同行1.txt`，例如 `《漏》_同行1.txt`。
- 如果第一份文件已经存在，后续文件依次命名为 `《课文名》-同行2.txt`、`《课文名》-同行3.txt`，选择当前未占用的最小序号。
- 若用户明确指定输出文件名或目录，优先遵从用户指定。

始终按以下结构输出：

### 课件基本信息

```markdown
- 文件名：
- 文件格式：
- 总页数：
- 可识别课文名称：
- 页序依据：
- 提取完整性说明：
```

### 逐页文本大纲

对每一页完整输出，不省略页码，不将连续页面合并：

```markdown
### 第 X 页
**页面标题：** [可识别标题；没有标题则写“无明确标题”]

**页面文字原文：**
[按页面可判断阅读顺序转写实际显示文字；无文字则写“未识别到可见文字”]

**页面类型：** [类型]

**提取方式：** [原生文本 / OCR识别，需核对 / 原生文本 + OCR补充，需核对]

**识别备注：** [文字完整 / 存在无法确认文字 / 关键内容建议以教材核对]

**画面或版式简述：** [只写可参考的页面组织或画面信息]
```

### 整体结构摘要

在忠实逐页转写完成后，简要输出：

- 总页数及课时划分线索；不能确定时写明无法确定。
- 页面组织顺序和主要教学环节。
- 重点覆盖的教学内容。
- 出现的课堂活动或练习类型。
- 可供后续原创大纲借鉴的组织方式。
- 需要谨慎核对、不宜直接复用或可能属于同行独特表达的内容。

### 交给 `ppt-dagang` 的参考资料说明

最后说明该提取结果可与教材、教师用书及教学设计资料一并提供给 `$ppt-dagang`。强调：

- 同行课件仅用于分析覆盖范围、教学环节和展示思路。
- 原创大纲中的事实、课文引用、教学目标和重难点应以权威资料为准。
- 对本次标记为 OCR 或无法确认的页面，应在引用前人工核查原页。

## 忠实性规则

- 保持页面边界，一页一节，不跨页合并。
- 尽量保留页面文字原字原序；只在阅读顺序确实无法判断时注明。
- 不纠正同行课件中的疑似错字，不自动填补残缺文字；可在备注中提示疑似问题。
- 不将同行文案优化、扩写或替换为新的页面文案。
- 不因页面重复而略过；重复页照常记录，并可在备注中指出重复。
- 不声称 OCR 结果百分之百准确。

## 质量检查

交付前核查：

- 页数是否与输入文件页数相符。
- 是否每页都有独立页码、原文、类型、提取方式与识别备注。
- 是否对图片页、乱码页、疑似漏字页做了 OCR 或明确说明无法完成。
- 是否将课件结构总结与原始逐页文本区分开。
- 是否明确该结果仅供 `$ppt-dagang` 参考而非直接复用。
- 最终交付文件是否为 `.txt`，且是否按照 `《课文名》_同行1.txt` / `《课文名》-同行N.txt` 的递增规则命名。
- 是否已将最终大纲保存到输入课件所在文件夹，或在无法写入/无法确定来源文件夹时明确记录原因并请用户确认。

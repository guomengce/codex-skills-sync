# Yuwen PPT 0615 Skill

## 适用场景

当用户提供以下任意一种或多种资料时，使用本 skill：

- 课文原文
- 课后题
- 教参
- 教学设计与指导
- 已审核通过的 PPT 大纲
- 已生成的纯图版 PPTX / 图文版 PPTX

目标是为小学语文课件建立一条稳定、少绕路、可复用、可审核、可编辑的制作流程。

## 总体原则

1. 先文本，后出图，再做可编辑叠加。
2. 先审核 `00`，再批量生成后续文件。
3. 页面风格必须明亮、干净、友好、适合二年级小学生审美，不使用阴暗压抑、灰脏浑浊、低质粗糙的画风。
4. 所有输出优先服务于“最终可编辑 PPT”，不是为了中间产物而增加流程复杂度。
5. 一旦用户已经在课文文件夹中放入新的 `纯图.pptx`、`图文.pptx`，后续执行必须以当前文件夹内的实际文件为准，不盲信旧版文档。

## 默认输出位置

默认输出到：

`D:\項目\999 program\yuwen-ppt`

用户通常会在此目录下按“年级 / 册数 / 单元 / 课文”建立文件夹。执行前应先确认当前课文所在目录，再在该目录内按固定序号写入产物。

## 固定流程

### 第一阶段：生成并审核 `00.PPT大纲.md`

输入资料来源可以包括：

- 课文原文
- 课后题
- 教参
- 教学设计与指导

本阶段只生成一个文件：

- `00.PPT大纲.md`

要求：

1. 按两课时设计，总页数约 40 页。
2. 先保证教学逻辑正确，再考虑页面好不好看。
3. 课文精讲部分必须尽量遵循课文顺序推进，不能随意打散。
4. 要符合教参与教学设计指导的要求。
5. 用 Markdown 生成，便于用户审核。
6. 层级清楚，避免正文信息和结构标记混在一起导致难审。

#### `00.PPT大纲.md` 通用生成规则

生成 `00` 时，先依据教参、教学设计和课后题确定课时结构，不机械规定每课时都必须包含同一套板块。若资料明确要求第一课时集中识字扫清障碍，就将主要字词学习放在第一课时；第二课时通常只做复习回顾，除非教参或教学设计另有安排。

两课时总页数约 40 页，并尽量均衡为每课时约 20 页。页数均衡要服从教学逻辑，不能为了凑页数打乱课文顺序、教学重点或课时目标。

尊重用户已有的大情境、主题包装和版面风格。可以优化教学逻辑、页面顺序和互动方式，但不要把某一课的大情境写成通用模板，也不要随意替换用户已确定的主题。

字词模块按教学资料确定位置和比重。通常在第一课时集中完成生字带拼音总览、识字方法、词语认读和图片理解，帮助学生扫清初读障碍。识字方法可根据本课特点选用偏旁归类、字理识字、动作识字、生活识字、多音字辨读、易错音提醒等，不要为每一种方法强行单独做页。

词语学习以教材词语表、教学目标和教学设计为准。重点呈现并学习词语表中的词语；课后积累类短语或拓展词语是否单独成页，要看教参和教学设计是否强调。若只需积累，可放入相关课文理解页中顺带出现，不强行升格为核心词语模块。

书写指导按教学设计分布，不设置固定的“书写评价”页。若教学设计将写字分到两个课时，就两个课时都安排书写指导；若教学设计集中安排，就按资料执行。书写指导页重点呈现结构、笔顺、关键笔画、易错点和范写观察。

顺文讲读必须尽量按课文顺序推进。每一部分围绕课时目标、课后题和教学重点设计问题、朗读、表达或练习，不为了视觉效果或互动形式随意打散课文理解链条。

`00` 阶段保持可审核，只写页码、标题、版面视觉/互动设计、展示文本和必要教学目的。不要提前写过细的文本框参数；后续 `01 / 02 / 03 / 04` 再细化可编辑文本层、背景图提示词和图文合成提示词。

`00.PPT大纲.md` 是后续 `01 / 02 / 03 / 04` 的唯一上游依据。只有在用户审核通过后，才能进入第二阶段。

### 第二阶段：根据 `00` 生成 4 个文件

本阶段固定生成：

- `01.PPT文本.md`
- `02.背景图提示词.md`
- `03.图文合成提示词.md`
- `04.提示词复制助手.html`

#### `01.PPT文本.md`

作用：作为“可编辑文本层”的主文档。

要求：

1. 与 `00` 逐页 1:1 对齐。
2. 明确每一页有哪些文本组件，例如：
   - 标题
   - 拼音
   - 正文
   - 提问
   - 练习
   - 词语卡片
   - 批注
   - 页内标签
3. 每个文本组件尽量写清：
   - 文字内容
   - 文本框数量
   - 文本框大致位置
   - 文本框宽高比例
   - 对齐方式
   - 字号层级
   - 颜色建议
   - 是否需要描边 / 底条 / 标签底板
4. 写法要服务于后续直接叠加可编辑文本框，而不是只写抽象描述。

#### `02.背景图提示词.md`

作用：生成纯背景图或纯图版 PPT 时使用。

要求：

1. 与 `01` 逐页对应。
2. 明确：
   - 整体场景
   - 主视觉元素
   - 文本安全区
   - 留白区域
   - 禁入区
3. 不写文本框本体，不让图片里直接生成可编辑文本框。
4. 重点是为后续叠加文字留出清晰、可读、稳定的位置。

#### `03.图文合成提示词.md`

作用：给浏览器里的 AI 生成图文版页面或图文版 PPT。

要求：

1. 与 `01 / 02` 逐页对应。
2. 风格、版面结构、文本摆放逻辑要与 `01 / 02` 一致。
3. 可以允许轻微发挥，但不能把整体结构改掉。
4. 目标是让图文版尽量接近未来的可编辑版视觉效果。

#### `04.提示词复制助手.html`

作用：方便用户在浏览器里逐页复制 `02 / 03` 的提示词。

要求：

1. 提供两个 tabs：
   - `02提示词`
   - `03提示词`
2. 每页一行。
3. 左边显示页码 / 标题，右边是复制按钮。
4. 支持 hover 高亮。
5. 点击后显示“已复制”反馈。
6. 页面不要太长，不展示大段正文，只服务复制。
7. 文件名前也保留固定序号，便于和其他产物一起排序查看。

### 第三阶段：以 `纯图.pptx` 为底稿，参考 `图文.pptx`，直接叠加可编辑文本层

这是最终成品阶段，也是本 skill 最重要的执行路线。

本阶段不再默认生成单独的“文本层 PPTX”，也不再默认走“先生成文本版再合并”的路线。  
标准做法是：直接在 `纯图.pptx` 上新增可编辑文本对象，让它本身变成可编辑课件。

#### 第三阶段输入

用户会把以下文件放回当前课文目录：

- `01.PPT文本.md`
- `纯图版.pptx`
- `图文版.pptx`

其中：

- `纯图版.pptx`：作为最终可编辑课件的底稿
- `图文版.pptx`：作为当前版本的视觉真值参考
- `01.PPT文本.md`：作为文字内容和组件结构依据

#### 第三阶段版本优先级

发生冲突时，按以下顺序判断：

1. 当前课文文件夹内最新的 `图文版.pptx`
2. 当前课文文件夹内对应的 `纯图版.pptx`
3. 当前版本的 `01.PPT文本.md`
4. 更早轮次生成的历史文档

也就是说：

- 页面视觉样式、文字排布、标题大小、标签样式、底板样式，以 `图文版.pptx` 为准。
- 页面底图、场景底板，以 `纯图版.pptx` 为准。
- 页面文字内容、文本框类别、文本组件结构，以 `01.PPT文本.md` 为准。

如果 `01` 与 `图文版.pptx` 明显不一致，不能闭眼照抄旧 `01`。应先按 `图文版.pptx` 核对当前页的真实结构，再回写或修正文本层。

#### 第三阶段推荐工具

优先使用以下工具组合：

1. Python 3.11
2. `python-pptx`
3. Tesseract OCR
4. Pillow
5. 必要时配合人工截图核对

推荐原因：

- `python-pptx` 适合直接在现有 PPTX 页面上增删文本框、设置字体、段落、边距、颜色、对齐、圆角底板等。
- Tesseract OCR 适合从 `图文版.pptx` 页面截图中提取已有文字，帮助判断文本内容、行数、文本块位置。
- Pillow 适合对截图做裁切、预处理、定位辅助。

#### 第三阶段标准执行流程

执行时尽量按以下顺序，不要跳步：

##### 1. 以 `图文版.pptx` 作为视觉样式参考源

这一步不是把整页图当成最终结果贴回去，而是把它当“样式参考图”。

要观察和提取的信息包括：

- 标题位置
- 拼音位置
- 主文本块位置
- 文本框尺寸比例
- 行数与换行节奏
- 字体大小层级
- 字体颜色
- 是否有描边
- 是否有半透明底板
- 底板圆角、描边、阴影、边距
- 词卡、标签、页签、编号等小组件的相对位置

核心目标不是“拿图顶掉可编辑层”，而是“把图里的版式重新用可编辑对象复刻出来”。

##### 2. 必要时对 `图文版.pptx` 页图做 OCR / 截图分析

当页面文本较多、样式较密、仅靠肉眼难以稳定复刻时，执行：

1. 将 `图文版.pptx` 对应页面导出或截图
2. 用 Tesseract OCR 提取文字和大致区域
3. 用 Pillow / 脚本辅助确定文本块位置
4. 再回到 PPT 中按可编辑对象重建

OCR 在这里的作用不是直接产出最终内容，而是辅助定位：

- 识别哪一块是标题
- 哪一块是正文
- 哪一块是标签
- 大致行高与占位范围

##### 3. 对照 `01.PPT文本.md` 建立可编辑文本对象

根据 `01` 中的文本内容和组件结构，在 `纯图版.pptx` 的对应页上叠加：

- 标题文本框
- 拼音文本框
- 正文文本框
- 标签底板与标签文字
- 练习题文本框
- 词语卡片
- 强调词
- 批注文字

注意：

1. 文本必须是可编辑文本框，不可把带字图片整页盖回去。
2. 只重建文字与文字承载组件，不改动纯图底图。
3. 如页面中需要底条、圆角白底、半透明信息板，也应用 PPT 形状 + 文本重建，而不是截图粘贴。

##### 4. 样式复刻原则

执行复刻时，优先追求以下四项一致：

1. 位置一致
2. 大小一致
3. 层级一致
4. 视觉气质一致

具体包括：

- 文本框中心点尽量对齐
- 文本块宽度和高度尽量一致
- 字号接近图文版实际观感
- 行距、段前后距、左右边距尽量一致
- 标题、正文、标签、拼音之间的层级关系一致
- 色彩和底板尽量一致

#### 第三阶段明确禁止的路线

为避免走弯路，以下路线默认不要再走：

1. 不要先单独生成一份“文本层 PPTX”再尝试和纯图合并。
2. 不要把图文版整页图片直接覆盖到可编辑课件里。
3. 不要只参考旧版 `01` 而忽略当前目录里的 `图文版.pptx`。
4. 不要为了省事把文字也烘焙进图片里。

#### 第三阶段默认输出

默认直接修改当前课文目录中的 `纯图版.pptx`，让它变成最终可编辑版本。  
只有用户明确要求保留副本时，才另存一个新文件。

## 执行口径总结

可以把这套 skill 理解为 3 句话：

1. 先根据资料生成并审核 `00.PPT大纲.md`
2. 再根据 `00` 生成 `01 / 02 / 03 / 04`
3. 最后以 `图文版.pptx` 为视觉参考，以 `01` 为文本依据，直接在 `纯图版.pptx` 上叠加可编辑文本层

这样做的目的只有一个：  
少走弯路，直接通向“可编辑、好看、能改、和图文版一致”的最终课件。
